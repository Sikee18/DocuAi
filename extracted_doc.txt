 RAG Chatbot Application (Reflex + LangChain) 
 1. Project Overview
This project is a Retrieval-Augmented Generation (RAG) based AI chatbot built using:
Reflex → Frontend + State Management
LangChain → Document processing & retrieval
ChromaDB → Vector database
LLM (Groq / Llama) → Answer generation
 2. Architecture of the Project
User → Upload Page → Documents Folder → RAG Backend
     → Vector Store → Retriever → LLM → Answer
Frontend (Reflex Pages)
    ↓
State Management (ChatState)
    ↓
Backend (rag.py)
 Flow
User uploads documents
Files are stored in /documents
Backend processes documents → creates embeddings
Vectorstore + Retriever created
User asks question
Retriever fetches relevant chunks
LLM generates answer using:
Context (documents)
Conversation history
Answer displayed with sources

 4. Why We Use State (IMPORTANT)
 Purpose of ChatState
Reflex uses State to:
Store user input
Store chat history
Handle uploads
Connect UI ↔ backend logic
 In Our App:
ChatState = Brain of the app
It manages:
question → current input
history → session memory
uploaded_files → uploaded docs
handle_upload() → file logic
ask() → chat logic
 5. Backend (rag.py) — Key Concepts
 Why RAG?
Instead of LLM guessing:
LLM + Documents → Accurate answers
 Key Components
1. Document Loading
Uses DirectoryLoader
Supports multiple files (PDF + TXT)
2. Chunking
Breaks documents into smaller parts
Helps better retrieval
3. Embeddings
Converts text → vectors
Used for similarity search
4. Vector Store (Chroma)
Stores embeddings
Enables fast retrieval
5. Retriever
Fetches top relevant chunks
6. Corrective RAG (Advanced Feature)
We added:
Relevance check (YES/NO)
Query rewriting if needed
 Important Backend Improvements We Did
1. Session-based Memory
Removed persist_directory
Vectorstore exists only during session
 2. Dynamic Document Addition
If vectorstore exists → add_documents()
Else → create new
 3. Conversation Memory (VERY IMPORTANT)
We modified:
corrective_rag(query, history)
Now LLM sees:
Past Q/A
Current query
4. History Injection into Prompt- final prompt
Conversation History + Context + Question
This allows:
Follow-up questions
Better answers

 7. State Logic (rag_state.py)
 ask()
Calls backend
Stores result
Updates history
 handle_upload()
Saves files
Runs RAG pipeline
Redirects to chat
reset_session()
Clears documents
Clears vectorstore
Clears retriever
8. Important Design Decisions
Why no persist_directory?
Because:
We want session-based memory (temporary)
 Why global vectorstore?
To:
Reuse same memory
Add new documents dynamically
 Why pass history manually?
Instead of LangChain memory:
Simpler
More control
Works perfectly for this project
